# toxic-bot

Telegram-бот для групових чатів з токсичною особистістю в дусі Венздей Аддамс.
Відповідає холодним сарказмом, логічним тиском і психологічною точністю —
на базі Groq LLM через шлюз Cloudflare AI Gateway.

## Можливості

- Відповідає в групових чатах з налаштовуваною частотою та рівнем токсичності
- Виявляє логічні помилки та психологічну слабкість у повідомленнях
- Відповідає гострим сарказмом — ніколи просто образами
- Налаштування на кожен чат через інлайн-клавіатуру (тільки для адмінів)
- Історія розмов у PostgreSQL
- Фонова сумаризація для економії контекстного вікна
- Багатомовний інтерфейс: англійська, російська, українська
- Автоматична реєстрація чатів при першому повідомленні
- PM-сповіщення суперадміну та система розсилки
- Інтеграція з Cloudflare AI Gateway для моніторингу та rate limiting
- Глобальний обробник помилок — без тихих падінь
- Автоматичний retry при flood control від Telegram

## Конфігурація

### `.env`

```dotenv
# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token_here

# Groq
GROQ_API_KEY=your_groq_api_key_here

# Cloudflare AI Gateway
CF_ACCOUNT_ID=your_cloudflare_account_id
CF_GATEWAY_ID=your_gateway_id

# PostgreSQL
DATABASE_URL=postgresql://user:password@localhost:5432/toxicbot

# ID суперадмінів у Telegram (через кому)
SUPERADMIN_IDS=123456789,987654321

# Виставляється автоматично після setup_commands.py — не редагувати вручну
# COMMANDS_REGISTERED=1
```

### `config.ini`

Всі значення опціональні — показані дефолти:

```ini
[bot]
max_history_messages = 20
default_lang         = en

[defaults]
toxicity_level     = 3
freq_min           = 5
freq_max           = 15
reply_cooldown_sec = 60
reply_chain_depth  = 5
min_words          = 5

[groq]
model       = moonshotai/kimi-k2-instruct-0905
temperature = 0.85
max_tokens  = 1024
top_p       = 0.95

[summarizer]
model       = llama-3.3-70b-versatile
max_tokens  = 512
temperature = 0.3
```

## База даних

Міграції запускаються автоматично при кожному старті через `db/migrations.py`.
Ручний SQL не потрібен. Таблиці що створюються:

| Таблиця         | Призначення                                              |
|-----------------|----------------------------------------------------------|
| `chat_settings` | Налаштування на кожен чат (токсичність, частота, мова)   |
| `history`       | Історія повідомлень розмови на кожен чат                 |
| `chats`         | Трекінг членства (додано, активний, кікнуто)             |

Чати реєструються автоматично при першому вхідному повідомленні — навіть групи,
додані до появи lifecycle-трекінгу, з'являться в `/sa_chats` після першої активності.

## Запуск

### Розробка

```bash
source .venv/bin/activate
python bot.py
```

### systemd (продакшен)

```ini
[Unit]
Description=Toxic Wednesday Telegram Bot
After=network.target postgresql.service

[Service]
User=youruser
WorkingDirectory=/home/youruser/toxic-bot
EnvironmentFile=/home/youruser/toxic-bot/.env
ExecStart=/home/youruser/toxic-bot/.venv/bin/python bot.py
Restart=always
RestartSec=5

[Install]

```

```bash
sudo systemctl enable toxic-bot
sudo systemctl start toxic-bot
journalctl -u toxic-bot -f
```

## Команди

### Публічні (всі чати)

| Команда          | Де          | Опис                                              |
|------------------|-------------|---------------------------------------------------|
| `/start`         | ОС + групи  | Привітання від бота                               |
| `/help`          | ОС + групи  | Інструкція з використання                         |
| `/about`         | ОС + групи  | Поточні налаштування особистості для цього чату   |
| `/reset`         | ОС + групи  | Видалити свою особисту історію розмови            |
| `/toxicity_demo` | ОС + групи  | Демо всіх п'яти рівнів токсичності                |

### Для адмінів (тільки адміни групи)

| Команда     | Де    | Опис                                                  |
|-------------|-------|-------------------------------------------------------|
| `/toxic`    | Групи | Реплай на повідомлення — змусити бота відповісти на нього |
| `/settings` | Групи | Відкрити інлайн-меню налаштувань                      |

Обидві команди мовчки видаляються якщо той хто викликає не є адміном —
ніяких публічних повідомлень "тільки для адмінів".

## Функції суперадміна

Суперадміни визначаються за Telegram user ID у `SUPERADMIN_IDS`.
Всі команди суперадміна працюють тільки в **особистому листуванні** з ботом.
Звичайні користувачі не бачать ці команди в меню і не отримують відповіді.

### Команди

| Команда          | Опис                                                  |
|------------------|-------------------------------------------------------|
| `/sa_chats`      | Список активних чатів з ID та датою додавання         |
| `/sa_stats`      | Статистика: всього / активних / неактивних за типами  |
| `/sa_broadcast`  | Розсилка повідомлення в усі активні групи (діалог)    |
| `/cancel`        | Перервати розсилку в процесі                          |

### Автоматичні сповіщення в ОС

Бот пише кожному суперадміну коли:

- **Доданий до чату** — назва, chat ID, тип
- **Кікнутий з чату** — назва, chat ID, тип

### Процес `/sa_broadcast`

```
Ти:  /sa_broadcast
Бот: Send the message to broadcast to all active group chats.
     Plain text only. /cancel to abort.
Ти:  Всім привіт, бот щойно оновився!
Бот: Broadcast complete.
     Sent: 3  Failed: 0
```

## Меню налаштувань

Адміни групи відкривають `/settings` для налаштування бота через інлайн-клавіатуру.
Всі зміни набувають чинності негайно без перезапуску.

| Налаштування       | Значення            | Опис                                               |
|--------------------|---------------------|----------------------------------------------------|
| Рівень токсичності | 1 – 5               | 1 = легкий сарказм, 5 = повна психологічна війна   |
| Частота відповідей | налаштовується      | Як часто бот вступає в розмову                     |
| Кулдаун            | 30 / 60 / 120 / 300 сек | Мінімальний час між відповідями бота           |
| Глибина ланцюжка   | 3 / 5 / 7 / 10      | Як глибоко в ланцюжку реплаїв стежить бот          |
| Мінімум слів       | 3 / 5 / 7 / 10      | Ігнорувати повідомлення коротші за це              |
| Мова               | en / ru / uk        | Мова системних повідомлень та меню бота            |

## Система особистості

Бот поводиться як Венздей Аддамс, але більш аналітично токсично:

- **Логічний тиск** — знаходить слабкі аргументи і розбирає їх точно
- **Психологічне прицілювання** — виявляє невпевненість, самовпевненість і суперечності
- **Дотепність замість образ** — кожна відповідь має сенс, а не просто шум
- **Контекстуальність** — використовує історію розмови для розвитку взаємодії
- **Реальні знання** — може і буде цитувати реальні факти для підсилення своєї точки зору

Рівні токсичності:

| Рівень | Поведінка                                               |
|--------|---------------------------------------------------------|
| 1      | Сухий гумор, легке зневажання                           |
| 2      | Помітне нетерпіння до твоєї логіки                      |
| 3      | Активний розбір твоїх аргументів (за замовчуванням)     |
| 4      | Особисте, хірургічне, складно ігнорувати                |
| 5      | Повна Венздей — холодно, точно, трохи лячно             |

## Структура проєкту

```
toxic-bot/
├── bot.py                    # Точка входу, реєстрація хендлерів, обробник помилок
├── config.py                 # Завантажувач конфігу (dataclasses + .env + config.ini)
├── config.ini                # Несекретні дефолти
├── .env                      # Секрети (не комітити)
├── .env.example              # Шаблон
├── setup_commands.py         # Одноразова реєстрація команд Telegram
├── requirements.txt
├── scripts/
│   └── backfill_chats.py     # Одноразова реєстрація вже існуючих чатів
├── ai/
│   ├── client.py             # Async Groq клієнт через Cloudflare AI Gateway
│   ├── responder.py          # get_reply() — основний LLM-виклик з історією
│   ├── prompts.py            # Будівник системного промпту за рівнем + мовою
│   └── summarizer.py         # Фонова сумаризація розмови
├── db/
│   ├── pool.py               # asyncpg connection pool
│   ├── migrations.py         # Ідемпотентні DDL-міграції, запуск при старті
│   ├── chat_settings.py      # CRUD налаштувань на кожен чат
│   ├── history.py            # CRUD історії розмови
│   └── chats.py              # CRUD трекінгу членства + статистика
├── handlers/
│   ├── commands_public.py    # /start /help /about /reset /toxicity_demo /toxic
│   ├── messages.py           # Основний хендлер повідомлень з логікою частоти і кулдауну
│   ├── lifecycle.py          # Події додавання/видалення + PM-сповіщення суперадміну
│   ├── superadmin.py         # /sa_chats /sa_stats /sa_broadcast
│   └── admin_menu/
│       ├── callbacks.py      # Константи callback data
│       ├── main_menu.py      # Точка входу інлайн-меню налаштувань
│       ├── simple_choice_menus.py  # Меню кулдауну / глибини ланцюжка / мін. слів
│       └── router.py         # Єдиний диспетчер CallbackQueryHandler
├── i18n/
│   └── __init__.py           # get_text(key, lang) — всі рядки інтерфейсу
└── utils/
    ├── admin_check.py        # is_chat_admin() з обходом для суперадмінів
    ├── rate_limiter.py       # Трекер кулдауну на користувача
    └── reply_chain.py        # Збирач контексту ланцюжка реплаїв
```
